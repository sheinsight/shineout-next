分析当前分支的所有代码变更，站在组件库使用者的视角，生成一段 Jira 单的需求/缺陷描述。

步骤：
1. 通过 git log --oneline 找到当前分支从基础分支分叉出来后的所有提交
2. 用 git diff 查看完整改动
3. 用**用户视角**描述问题，不要提及内部实现细节（如变量名、函数名、文件路径）

输出格式：

**标题：** 一句话概括问题或需求

**描述：**
简要说明场景和问题现象

**期望行为：**
用户期望组件如何表现

**实际行为：**
当前组件实际的表现（如果是 bug fix）

**需求背景**
描述当前遇到了什么问题，如果不解决会有什么影响

**价值说明**
1、成本；2、提高稳定性；3、提升安全性；4、提高效率；5、能力补充

要求：
- 语言简洁，避免技术术语，让非开发人员也能看懂
